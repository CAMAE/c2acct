# Module 3: Assurance of Learning and Institutional Design

## Position

This module is part of an internal professional-learning package. It is designed to be CPE-ready in architecture and accreditation-aligned in structure, but it does not claim approved CPE credit.

## Field of Study Focus

Management Services / Specialized Knowledge

## Module Purpose

Frames rigorous modular learning using AACSB and university models instead of lightweight engagement mechanics.

## Learning Objectives

- Explain why learning goals and assessment evidence belong together.
- Connect stackable learning design to serious professional education.
- Map module, quiz, and final layers to an assurance-of-learning model.

## Key Topics

### AACSB measurable outcomes
- Principle: AACSB-style learning design expects measurable outcomes, direct assessment, and continuous improvement.
- Operational implication: Define module objectives before finalizing questions, unlocks, or completion thresholds.
- Required evidence artifact: A documented objective set tied to quiz and final-test evidence.
- Primary failure risk: Learners can finish modules without a defensible statement of what was actually learned.
- Why it matters: Outcomes and evidence must be linked if the program is meant to withstand institutional scrutiny.
- Source: SRC03 | AACSB Business Accreditation Standards | https://www.aacsb.edu/accredited/standards/business

### Rutgers accounting learning goals
- Principle: Serious accounting learning models explicitly include communication, quantitative skill, ethics, and information-technology capability.
- Operational implication: Use multi-competency objectives instead of treating the curriculum as a single generic topic bucket.
- Required evidence artifact: A coverage map showing which competencies each module and quiz addresses.
- Primary failure risk: The program appears narrow, under-specified, or unbalanced.
- Why it matters: Professional learning becomes more credible when it reflects recognizable accounting competencies.
- Source: SRC10 | Rutgers Accounting Learning Goals | https://myrbs.business.rutgers.edu/undergraduate-newark/accounting-minor

### Stackable module architecture
- Principle: Stackable design is credible when smaller learning units accumulate into a governed larger outcome.
- Operational implication: Require module completion, checkpoint quizzes, and a cumulative final before issuing a completion record.
- Required evidence artifact: A progression map showing which completions unlock later stages.
- Primary failure risk: End-state recognition is issued without the evidence stack beneath it.
- Why it matters: A stack only works if each component is governed, scoped, and sequenced intentionally.
- Source: SRC11 | Rutgers Stackable Program | https://www.business.rutgers.edu/stackable

### Assessment and competencies
- Principle: Student learning outcomes and competencies should be visible in the assessment design, not hidden behind completion counts.
- Operational implication: Tie each question cluster to a documented competency or field-of-study rationale.
- Required evidence artifact: A source map and rationale set that explains why each question is present.
- Primary failure risk: The question bank looks broad but cannot defend its educational purpose.
- Why it matters: Assessment quality depends on fit between questions and intended outcomes.
- Source: SRC12 | Kelley Accounting Assessment | https://kelley.iu.edu/about/directory/offices/instructional-consulting/assessment/accounting.html

### Improvement-loop governance
- Principle: An accreditation-aligned learning system needs review loops, not static content dumps.
- Operational implication: Version modules and review outcomes so questions, thresholds, and source maps can be revised responsibly.
- Required evidence artifact: A revision-aware content package with dated source references and version identifiers.
- Primary failure risk: Outdated content remains live with no audit trail for what changed or why.
- Why it matters: Continuous improvement is only real when changes can be traced and justified.
- Source: SRC03 | AACSB Business Accreditation Standards | https://www.aacsb.edu/accredited/standards/business

## Assessment Structure

- Reading completion is required before the quiz is treated as eligible.
- This module contributes 25 source-backed question-bank items and 10 quiz items.
- Questions are tied to explicit topics rather than generic engagement prompts.
